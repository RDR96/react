# Configuring Custom Toast Notifications

You can customize toast notifications in the Payables, PayableDetails, Receivables, and ReceivableDetails components by using the `renderCustom` prop. This allows you to override the default toast implementation with your own notification system.

## Examples using Alert

The examples below use the basic `alert()` function, but you can implement any custom notification system of your choice, such as react-toastify, Material-UI Snackbar, or your own custom components.

Your custom toast should be configured with the following interface:

```ts
interface ToastClient {
  success: (message: string) => void
  error: (message: string) => void
  info: (message: string) => void
}
```

### Payables Component

<ComponentContainer showCodeTabFirst>
  <Payables
    handlers={{
      onSelectInvoice: (invoice) => router.push(`/dashboard/vendors/${mercoaSession.entityId}/invoices/${invoice.id}`),
    }}
    renderCustom={{
      toast: {
        success: (message) => alert(message),
        error: (message) => alert(message),
        info: (message) => alert(message),
      },
    }}
  />
  ```jsx
  <Payables
    handlers={{
      onSelectInvoice: (invoice) => {
        router.push(`/dashboard/vendors/${mercoaSession.entityId}/invoices/${invoice.id}`)
      },
    }}
    renderCustom={{
      toast: {
        success: (message) => alert(message),
        error: (message) => alert(message),
        info: (message) => alert(message),
      },
    }}
  />
  ```
</ComponentContainer>
